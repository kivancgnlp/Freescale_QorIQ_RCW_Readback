# QorIQ RCW Reader

A browser-based tool for parsing NXP QorIQ PBL boot images and decoding Reset Configuration Word (RCW) fields. Useful for inspecting and verifying the RCW configuration of QorIQ processors without requiring any installed software.

**[Open the app](https://kivancgnlp.github.io/WebApps/Freescale_QorIQ_RCW_Readback)**

## What it does

- Parses a PBL binary file and extracts the 64-byte RCW payload
- Decodes each RCW field by name, bit position, raw value, and human-readable meaning
- Validates the PBL CRC32 (CRC32/MPEG-2) if an end command is present in the image
- Calculates derived clock frequencies (core, platform, DDR, etc.) from a user-supplied SYSCLK value and checks them against datasheet limits

## Supported processors

| Processor | Notes |
|-----------|-------|
| P3041     | 4-core QorIQ |
| T2080     | 8-core QorIQ |
| T2081     | 4-core T2080 variant — identical RCW layout |

## Input file format

The tool expects a **PBL (Pre-Boot Loader) binary** file (`.bin`), as generated by the NXP CodeWarrior or custom board bring-up tooling. The expected layout is:

| Offset | Size | Content |
|--------|------|---------|
| 0x00   | 4 B  | Preamble magic: `AA 55 AA 55` |
| 0x04   | 4 B  | RCW destination address (big-endian u32, e.g. `01 0E 01 00`) |
| 0x08   | 64 B | RCW payload (512 bits) |
| 0x48   | ...  | PBI instructions (optional) |
| —      | 4 B  | End command: `08 13 80 40` (if CRC is present) |
| —      | 4 B  | CRC32/MPEG-2 of all bytes up to and including the end command |

Files that do not start with `AA 55 AA 55` or are shorter than 72 bytes will be rejected with an error message.

## Usage

1. Open the [app link](https://kivancgnlp.github.io/WebApps/Freescale_QorIQ_RCW_Readback) in any modern browser — no installation required.
2. Select the target processor from the dropdown.
3. Drop or browse to your PBL `.bin` file.
4. Enter the board SYSCLK frequency in MHz (default: 100 MHz).
5. Click **Parse RCW**.

The RCW fields table and derived frequency cards will appear below. Fields whose raw value has no matching definition in the configuration are flagged with an **UNKNOWN** badge.
